Les Variables en JavaScript

Les variables sont des conteneurs pour stocker des données. Elles sont fondamentales dans tout langage de programmation.

// Déclaration d'une variable avec 'var' (ancien)
var age = 30;
console.log(age); // Affiche: 30
// Déclaration d'une variable avec 'let' (moderne, réassignable)
let nom = "Alice";
nom = "Bob"; // On peut réassigner 'nom'
console.log(nom); // Affiche: Bob
// Déclaration d'une constante avec 'const' (moderne, non réassignable)
const PI = 3.14159;
// PI = 3.14; // Ceci provoquerait une erreur: Assignment to constant variable.
console.log(PI); // Affiche: 3.14159

Types de Déclaration de Variables

1. var (à éviter)

C'est la méthode traditionnelle pour déclarer des variables en JavaScript. Cependant, elle présente des inconvénients comme le hoisting et le fait qu'elle est "function-scoped" et non "block-scoped", ce qui peut entraîner des comportements inattendus.

if (true) {
  var x = 10;
}
console.log(x); // Affiche 10, car 'var' n'est pas limité au bloc 'if'.

2. let (préféré)

let a été introduit avec ES6 (ECMAScript 2015) pour résoudre les problèmes de var. Les variables déclarées avec let sont block-scoped et ne sont pas soumises au même type de hoisting que var. Elles peuvent être réassignées.

if (true) {
  let y = 20;
  console.log(y); // Affiche 20
}
// console.log(y); // Erreur: y is not defined, car 'let' est block-scoped.

3. const (préféré pour les valeurs fixes)

const est également introduit avec ES6. Les constantes sont block-scoped et, une fois qu'une valeur leur est assignée, elles ne peuvent pas être réassignées. Cela ne signifie pas que la valeur est immuable, mais plutôt que la référence à cette valeur ne peut pas changer (important pour les objets et les tableaux).

const utilisateur = { nom: "Alice" };
utilisateur.nom = "Bob"; // Ceci est permis car l'objet lui-même est modifié, pas la référence.
console.log(utilisateur.nom); // Affiche: Bob
// utilisateur = { nom: "Charlie" }; // Erreur: Assignment to constant variable.

Portée (Scope) des Variables

Portée Globale

Une variable déclarée en dehors de toute fonction ou bloc a une portée globale et est accessible de partout dans le code.

var globale = "Je suis globale";
function montrerGlobale() {
  console.log(globale); // Accessible
}
montrerGlobale();

Portée de Fonction (Function Scope)

Les variables déclarées avec var à l'intérieur d'une fonction ont une portée de fonction. Elles ne sont accessibles qu'à l'intérieur de cette fonction.

function maFonction() {
  var locale = "Je suis locale";
  console.log(locale);
}
maFonction(); // Affiche: Je suis locale
// console.log(locale); // Erreur: locale is not defined

Portée de Bloc (Block Scope)

Les variables déclarées avec let et const à l'intérieur d'un bloc (délimité par des accolades {}, comme un if, for, ou while) ont une portée de bloc. Elles ne sont accessibles qu'à l'intérieur de ce bloc.

if (true) {
  let blockVar = "Je suis de portée bloc";
  const blockConst = "Moi aussi";
  console.log(blockVar); // Accessible
  console.log(blockConst); // Accessible
}
// console.log(blockVar); // Erreur: blockVar is not defined
// console.log(blockConst); // Erreur: blockConst is not defined
Bonne pratique : Utilisez toujours let ou const pour déclarer vos variables. Évitez var pour prévenir des erreurs courantes et améliorer la lisibilité du code.
